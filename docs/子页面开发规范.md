# 子菜单业务页开发规范

## 原则

- **所有子菜单落地页**（列表/表单类业务页）**视觉与交互与「仓库管理」页面一致**。
- **不要从零写布局**：以 **`warehouse.html`** 或 **`subpage-template.html`** 为蓝本复制，**只改内容区**（标题、筛选项、表格、脚本）。

## 必引样式

在 `<head>` 中保持与仓库管理相同：

```html
<link rel="stylesheet" href="css/subpage-toolbar.css" />
<link rel="stylesheet" href="css/subpage-base.css" />
```

- **`subpage-base.css`**：壳层、侧栏、主区白底、`.wh-*` 筛选区、表格、分页等**统一布局**。
- **`subpage-toolbar.css`**：主区「新增 / 修改 / 删除 / 导出」四按钮样式。

操作栏按钮用法见：**[子页面操作栏规范.md](./子页面操作栏规范.md)**。

## Body 类名

```html
<body class="page-subpage">
```

若某页需与旧代码兼容，可同时写 `page-subpage page-xxx`；**至少包含 `page-subpage`**，以便应用 `subpage-base.css` 中的全局样式。

## 新建页面步骤

1. **复制** `subpage-template.html`（或 `warehouse.html`）为新文件名，例如 `inbound.html`。
2. 修改 **`<title>`** 与主区 **`<h2 class="wh-title">`**。
3. 按需调整 **左侧导航高亮**（`nav-item--module-active` 等）与 **仓储二级面板** 中当前项（若该菜单从驾驶舱进入）。
4. 在 **`.wh-filters`** 中改筛选项的 label、`id`、`placeholder`。
5. 在 **表格** 中改 `<thead>` / `<tbody>` 列与示例数据。
6. 修改 **底部 `<script>`** 中的筛选、分页、导出等逻辑（保持 DOM 结构与 class 命名习惯，便于维护）。

## 不要改动的约定（除非全项目统一调整）

- 主区外层结构：`main.main-scroll`、`.wh-title`、`.wh-filters`、`.btn-toolbar`、`.wh-table-wrap`、分页容器等 **class 命名**。
- 四色操作按钮的 **class**：`btn-add` / `btn-edit` / `btn-delete` / `btn-export`（见操作栏规范文档）。
